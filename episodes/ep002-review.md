### 수정 필요 사항

1. [긴급] 분량 초과 -- 본문이 5,000자 목표 상한을 약 800~1,000자 초과. 다음 구간 압축 권장:
   - 103~106행: 미래 사건 나열이 6건. "부산 해운대 B급 브레이크 / 천랑 길드 비리 / 강현준 / 흑요석 유통망 / S급 붕괴"로 5건 중 1~2건을 삭제하거나, 3줄 이내로 압축. 독자는 아직 이 인물/사건을 모르므로 세부 정보는 의미가 희석됨.
   - 126~132행: 용산 포터 구역 거리 묘사가 과도. "아직 철거되지 않은 건물", "김씨 수선집", "식당", "모퉁이 포터" 중 1~2개만 남기고 압축.

2. [긴급] 시스템 UI 포맷 불일치 -- EP001과 동일 문제.
   - 상태 정보 창 (32~44행): v2.0 포맷 `┌─┐`로 변경
   - 제로 코드 창 (60~82행): v0.0 포맷 `╔═╗`로 변경

3. [긴급] 제로 코드 UI에 "삭제 영역 감지 (Deleted Area Scan) -- 권한 레벨 3 필요"라고 표기(73행)되어 있으나, EP004에서 권한 레벨 1 상태에서 삭제 영역이 자동 감지됨. system-ui.md의 제로 코드 상태 오버레이에는 "삭제 영역: 감지 가능 (접근 제한됨)"으로 되어 있어, "감지"와 "접근"을 구분해야 함. EP002의 잠긴 기능 목록을 "삭제 영역 접근 (Deleted Area Access) -- 권한 레벨 3 필요"로 수정 권장. 그래야 EP004에서 "감지는 되지만 접근은 못 하는" 상태와 일관성이 맞음.

4. [권장] 96~98행: 관리자 감지 확률 설명이 설정 직접 인용에 가까움("권한 레벨이 높아질수록 감지 확률도 기하급수적으로 증가한다"). 캐릭터 내면 독백으로 재구성하면 자연스러움. 예: "제로 코드를 쓸수록 관리자의 눈에 가까워진다. 지금은 바늘구멍만큼도 안 되는 틈이지만, 그 틈은 넓어질 것이다."

5. [권장] 클리프행어 강화 -- 현재 "이번엔 다르다."로 끝나는데, 백도현의 뒷모습을 보는 순간의 클리프행어(episode-plan.md 기준: "백도현의 뒷모습을 본다. 한시우의 눈이 아주 잠깐 흔들린다")가 에피소드 본문에는 있으나, 마지막 장면으로 배치되지 않고 중간(180~191행)에 위치. 실제 마지막 문장은 "이번엔 다르다"인데, 이것 자체는 나쁘지 않으나 백도현 조우를 마지막으로 재배치하면 더 강한 견인력 확보 가능.
